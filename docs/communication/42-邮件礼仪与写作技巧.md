# 邮件礼仪与写作技巧

## 本章概览

专业的邮件能提升个人形象和沟通效率。本章介绍商务邮件的规范、写作技巧和礼仪要点。

**学习目标**
- 掌握商务邮件规范
- 学会撰写清晰主题
- 掌握邮件结构组织
- 了解邮件礼仪要点

---

## 42.1 邮件基本规范

### 42.1.1 邮件组成

**完整邮件结构**
```
收件人: 主要接收者
抄送: 需要知悉的人
密送: 隐藏接收者(慎用)
主题: 清晰概括邮件内容
正文:
  ├─ 称呼
  ├─ 问候语
  ├─ 正文主体
  ├─ 结束语
  └─ 签名
附件: 相关文件(如有)
```

### 42.1.2 主题撰写

**主题原则**
```
清晰: 一眼看懂邮件内容
具体: 包含关键信息
简洁: 不超过30字
```

**主题格式**
```
工作邮件:
[类型] 具体内容 - 时间/项目

示例:
✓ [请批] 采购申请 - XX项目
✓ [通知] 周五下午部门会议
✓ [提交] 季度报告 - 2024Q1
✓ [请假] 年假申请 1月15-17日

❌ 避免:
❌ 重要
❌ 关于那个事
❌ 请查看
❌ (无主题)
```

**主题前缀**
```
[请批]: 需要批准
[通知]: 通知类
[提交]: 提交文件
[请假]: 请假申请
[会议]: 会议相关
[紧急]: 紧急事项(慎用)
[FYI]: 仅供参考
[回复]: 回复邮件(自动)
```

## 42.2 邮件正文写作

### 42.2.1 称呼与问候

**称呼规范**
```
正式场合:
尊敬的XX总/XX经理:
XX总您好:

一般场合:
XX您好,
各位同事,
Hi XX,

群发邮件:
各位同事/领导:
团队成员:
```

**问候语**
```
初次联系:
您好,我是XX公司的XX,很高兴认识您。

日常沟通:
您好,(直接进入正题)

特殊场合:
新年快乐!
节日愉快!
```

### 42.2.2 正文结构

**SCQA结构**
```
S(Situation): 背景情况
C(Complication): 遇到问题
Q(Question): 核心问题
A(Answer): 解决方案

示例:
【背景】本季度销售目标100万
【问题】目前仅完成60万,进度滞后
【疑问】如何在剩余时间达成目标?
【方案】建议增加促销活动,具体如下...
```

**金字塔原则**
```
结论先行:
先说结论/建议
再说理由/依据

示例:
❌ 差:
"经过调研发现...数据显示...因此建议..."

✓ 好:
"建议采用方案A。理由如下:1...2...3..."
```

### 42.2.3 内容组织

**段落分隔**
```
原则:
- 一段一个观点
- 段落间空行
- 避免大段文字

示例:
第一段: 目的说明

第二段: 核心内容1

第三段: 核心内容2

第四段: 总结/行动项
```

**列表使用**
```
有序列表(步骤/优先级):
1. 第一步...
2. 第二步...
3. 第三步...

无序列表(并列关系):
• 要点1
• 要点2
• 要点3

技巧:
不超过7条
每条简洁明了
```

## 42.3 结尾与签名

### 42.3.1 结束语

**常用结束语**
```
期待回复:
期待您的回复。
请您审阅并反馈。
烦请确认,谢谢。

表示感谢:
感谢您的支持!
谢谢您的时间。

一般结束:
如有疑问请随时联系。
祝工作顺利!
```

**行动号召**
```
明确下一步:
请于本周五前回复。
请查看附件并签字确认。
请参会人员回复是否参加。

避免模糊:
❌ 请处理
❌ 请关注
✓ 请在3月1日前完成审批
```

### 42.3.2 签名设置

**专业签名**
```
基础版:
---
张三
市场部 | 市场专员
XX科技有限公司
电话: 138-0000-0000
邮箱: zhangsan@company.com

完整版:
---
张三 Zhang San
市场部 市场专员
Marketing Specialist

XX科技有限公司 XX Technology Co., Ltd.
手机: +86 138-0000-0000
邮箱: zhangsan@company.com
地址: 北京市朝阳区XX路XX号
官网: www.company.com
```

**签名设置**
```
Outlook:
文件 → 选项 → 邮件 → 签名

创建签名:
1. 新建签名
2. 输入签名内容
3. 设置默认签名(新邮件/回复)

技巧:
- 不超过6行
- 字号略小于正文
- 可添加公司Logo(小尺寸)
- 避免花哨设计
```

## 42.4 收件人管理

### 42.4.1 收件人/抄送/密送

**正确使用**
```
收件人(To):
- 需要回复或采取行动的人
- 主要责任人

抄送(CC):
- 需要知晓但不需回复的人
- 相关方、领导(知会)

密送(BCC):
- 保护收件人隐私(群发)
- 特殊情况隐藏接收者

原则:
不要滥用抄送
谨慎使用密送
明确主送对象
```

**全部回复礼仪**
```
何时使用"全部回复":
✓ 信息需要所有人知道
✓ 回答所有人的疑问
✓ 公开讨论的问题

避免"全部回复":
❌ 仅与发件人相关
❌ 简单的"谢谢"、"收到"
❌ 私人话题
```

### 42.4.2 群发邮件

**群发注意**
```
使用密送:
保护收件人邮箱隐私
避免"回复全部"轰炸

个性化:
使用邮件合并功能
称呼使用姓名
避免明显群发感

示例:
❌ 亲爱的客户们
✓ 张总您好(邮件合并)
```

## 42.5 附件处理

### 42.5.1 附件规范

**附件命名**
```
格式:
文件类型_项目名称_版本_日期.扩展名

示例:
✓ 合同_XX项目_V1.0_20240115.pdf
✓ 报告_季度总结_2024Q1.docx
✓ 发票_202401.xlsx

❌ 避免:
❌ 新建文档.docx
❌ 副本 (2).xlsx
❌ 未命名.pdf
```

**附件说明**
```
正文提及:
"请查看附件中的XX文件"
"附件为本次会议的资料,共3个文件"

附件清单(多文件时):
附件:
1. 合同草稿.pdf
2. 报价单.xlsx
3. 产品手册.pdf
```

### 42.5.2 大附件处理

**解决方案**
```
压缩文件:
- 使用ZIP/RAR压缩
- 减小体积
- 多文件打包

云盘链接:
OneDrive/百度网盘链接
设置访问权限
注明有效期

分卷发送:
单个文件过大时
分多封邮件发送
正文说明顺序
```

## 42.6 邮件礼仪

### 42.6.1 回复时效

**回复时间**
```
紧急邮件: 1小时内
一般邮件: 24小时内
非工作时间: 下个工作日

暂时无法处理:
"您的邮件已收到,我将在X日前回复您详细信息。"

自动回复:
休假时设置自动回复
说明返回时间
提供紧急联系方式
```

### 42.6.2 语气与措辞

**礼貌用语**
```
请求:
请您...
能否麻烦您...
如果方便的话...

感谢:
感谢您的...
非常感谢...
谢谢配合

道歉:
抱歉...
给您带来不便...
非常抱歉...
```

**避免情绪化**
```
❌ 负面表达:
"你怎么还没做!"
"这个问题说了多少遍了"
"太差了"

✓ 专业表达:
"请问进度如何?"
"再次提醒..."
"建议改进..."
```

### 42.6.3 注意事项

**书写规范**
```
✓ 正确:
- 使用标准标点
- 检查错别字
- 语句通顺
- 段落清晰

❌ 避免:
- 全大写(视为吼叫)
- 过多感叹号!!!
- 表情符号过多😊😊😊
- 网络用语(666、hhh)
```

**隐私保护**
```
注意:
- 不要转发他人私密邮件
- 密送名单不泄露
- 敏感信息加密或单独沟通
- 离职时清理邮件
```

## 42.7 特殊场景邮件

### 42.7.1 请求邮件

**结构**
```
主题: [请批] 明确请求事项

正文:
1. 说明背景
2. 具体请求
3. 理由支持
4. 期待回复

示例:
XX领导您好,

因XX项目需要,需申请采购XX设备一台,
预算5000元。该设备将用于...

请您审批,谢谢。
```

### 42.7.2 通知邮件

**结构**
```
主题: [通知] 具体事项

正文:
- What: 什么事
- When: 什么时间
- Where: 什么地点
- Who: 涉及谁
- How: 如何参与

示例:
各位同事,

定于本周五下午3点在会议室A召开部门会议,
讨论下季度计划,请准时参会。

请回复确认是否参加。
```

### 42.7.3 道歉邮件

**结构**
```
主题: [道歉] 具体事项

正文:
1. 明确道歉
2. 说明原因
3. 补救措施
4. 保证改进

示例:
XX总您好,

非常抱歉,由于我的疏忽导致报告数据有误,
给您的决策带来困扰。

我已重新核对数据并更新报告(见附件),
今后会加强审核。再次致歉。
```

## 42.8 实战案例

### 案例1: 会议邀请邮件

**邮件示例**
```
主题: [会议] 产品发布会筹备会议 - 1月20日

各位同事,

定于1月20日(周五)下午2:00在3楼会议室召开
产品发布会筹备会议,讨论以下议题:

1. 发布会流程确认
2. 物料准备情况
3. 人员分工

请相关部门参会人员准时参加,并提前准备发言内容。

会议时长: 约1小时
请回复确认是否参加。

谢谢!

---
张三
市场部
```

【进行中】

李四

## 本章小结

**核心要点**
1. **主题**: 清晰具体,使用前缀
2. **结构**: 称呼+正文+结束语+签名
3. **内容**: 结论先行,条理清晰
4. **礼仪**: 及时回复,措辞礼貌
5. **附件**: 规范命名,说明清单

**写作原则**
- 清晰(Clear)
- 简洁(Concise)
- 完整(Complete)
- 礼貌(Courteous)
- 正确(Correct)

**检查清单**
- [ ] 主题清晰具体
- [ ] 收件人正确
- [ ] 称呼得体
- [ ] 内容条理清晰
- [ ] 附件完整
- [ ] 签名完整
- [ ] 检查错别字
